# UBORROWU · ระบบยืม-คืนพัสดุออนไลน์ มหาวิทยาลัยอุบลราชธานี — Use Case Description

> อ้างอิงจาก `uborrowu_functional_requirements.md` (v1.2) — 1 use case ในเอกสารนี้ต้องตรงกับ 1 use case ใน `usecase.puml` (UC-01 ถึง UC-35)
> ผู้ใช้งาน 3 บทบาท: 🎓 นักศึกษา (Student), 👷 เจ้าหน้าที่ (Staff), 👑 ผู้ดูแลระบบ (Admin)

**หมายเหตุทั่วไป (Cross-cutting concern):** ทุก use case ในเอกสารนี้ (ยกเว้น UC-16 การเข้าสู่ระบบเอง) มีเงื่อนไขก่อนเริ่มทำงานร่วมกันคือ **ผู้ใช้ต้องเข้าสู่ระบบด้วย Google OAuth 2.0 และผ่านการตรวจสอบสิทธิ์จาก Whitelist แล้ว (`login_required` + ตรวจสอบ role/permission ผ่าน django-allauth)** จึงไม่ระบุซ้ำในทุกแถว "URL/View + สิ่งที่ต้องพึ่งพา" ของแต่ละ use case ด้านล่าง

## Epic B · ตะกร้าและการยืม (Student)

#### UC-05 · จัดการตะกร้าสิ่งของ

| หัวข้อ | รายละเอียด |
|-------|-----------|
| **รหัสอ้างอิง** | FR-05 |
| **Actor หลัก** | นักศึกษา (Student) |
| **Actor รอง** | - |
| **คำอธิบายโดยย่อ** | นักศึกษาต้องการรวบรวมสิ่งของที่ต้องการยืมไว้ในตะกร้า ได้แก่ เพิ่มสิ่งของ แก้ไขจำนวน และลบออก ก่อนส่งคำขอยืมจริง |
| **เงื่อนไขก่อนเริ่มทำงาน (Pre-condition)** | นักศึกษาเข้าสู่ระบบแล้ว และสิ่งของที่ต้องการเพิ่มมีจำนวนคงเหลือมากกว่า 0 |
| **ลำดับเหตุการณ์หลัก (Basic Flow)** | 1. นักศึกษากดเพิ่มสิ่งของลงตะกร้าจากหน้ารายการ/รายละเอียดสิ่งของ<br>2. นักศึกษาเปิดหน้าตะกร้า<br>3. นักศึกษาแก้ไขจำนวนที่ต้องการยืมของแต่ละรายการ หรือลบรายการออกจากตะกร้า<br>4. ระบบบันทึกสถานะตะกร้าล่าสุด |
| **ลำดับเหตุการณ์ทางเลือก (Alternative/Exception Flow)** | หากจำนวนที่ต้องการเกินจำนวนคงเหลือจริง ระบบแจ้งเตือนและไม่อนุญาตให้เพิ่ม/แก้ไขเกินจำนวนนั้น |
| **เงื่อนไขหลังทำงานสำเร็จ (Post-condition)** | ตะกร้ามีรายการสิ่งของและจำนวนที่นักศึกษาต้องการยืมพร้อมส่งคำขอ |
| **ความสัมพันธ์กับ Use Case อื่น** | ถูก `<<include>>` โดย UC-06 (ต้องมีสิ่งของในตะกร้าก่อนส่งคำขอยืมเสมอ) |
| **URL/View + สิ่งที่ต้องพึ่งพา** | `borrow:cart_detail`, `borrow:cart_add`, `borrow:cart_update`, `borrow:cart_remove` → `CartDetailView`, `CartAddView`, `CartUpdateView`, `CartRemoveView` (อ่านจำนวนคงเหลือจาก model ของ `catalog`) |

#### UC-06 · ส่งคำขอยืมสิ่งของ

| หัวข้อ | รายละเอียด |
|-------|-----------|
| **รหัสอ้างอิง** | FR-06 |
| **Actor หลัก** | นักศึกษา (Student) |
| **Actor รอง** | - |
| **คำอธิบายโดยย่อ** | นักศึกษาต้องการส่งคำขอยืมสิ่งของหลายชิ้นพร้อมกันในคราวเดียว พร้อมระบุเหตุผลการยืม (และวันที่ต้องการใช้งานสำหรับครุภัณฑ์) เพื่อรอเจ้าหน้าที่อนุมัติ |
| **เงื่อนไขก่อนเริ่มทำงาน (Pre-condition)** | นักศึกษามีสิ่งของอยู่ในตะกร้าอย่างน้อย 1 รายการ, ข้อมูลติดต่อของนักศึกษาต้องกรอกครบถ้วนแล้ว (ตาม FR-15), นักศึกษาต้องไม่ถูกระงับสิทธิ์การยืม (Block) |
| **ลำดับเหตุการณ์หลัก (Basic Flow)** | 1. นักศึกษากดยืนยันส่งคำขอยืมจากหน้าตะกร้า<br>2. นักศึกษาระบุเหตุผลการยืม และวันที่ต้องการใช้งาน (กรณีมีครุภัณฑ์อยู่ในรายการ)<br>3. ระบบตรวจสอบข้อมูลติดต่อและสถานะการระงับสิทธิ์ของนักศึกษา<br>4. ระบบบันทึกคำขอยืมเป็นสถานะ "รออนุมัติ" และล้างตะกร้า |
| **ลำดับเหตุการณ์ทางเลือก (Alternative/Exception Flow)** | หากข้อมูลติดต่อไม่ครบ ระบบพาไปหน้าแก้ไขโปรไฟล์ก่อน (เชื่อมโยง UC-14)<br>หากนักศึกษาถูกระงับสิทธิ์อยู่ ระบบปฏิเสธการส่งคำขอและแจ้งเหตุผล<br>หากไม่ระบุวันที่ใช้งานสำหรับครุภัณฑ์ ระบบแจ้งเตือนให้กรอกให้ครบ |
| **เงื่อนไขหลังทำงานสำเร็จ (Post-condition)** | มีคำขอยืมสถานะ "รออนุมัติ" ถูกสร้างขึ้น รอเจ้าหน้าที่พิจารณา (UC-18) |
| **ความสัมพันธ์กับ Use Case อื่น** | `<<include>>` UC-05 (ต้องมีตะกร้าก่อนเสมอ); เป็นฐานของ UC-18 (เจ้าหน้าที่อนุมัติ/ปฏิเสธ) |
| **URL/View + สิ่งที่ต้องพึ่งพา** | `borrow:request_create` → `RequestCreateView` (ตรวจสอบข้อมูลโปรไฟล์และสถานะ block จาก model ผู้ใช้) |
