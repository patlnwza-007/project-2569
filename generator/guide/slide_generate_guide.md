# คู่มือการจัดวางเนื้อหาสไลด์ (Slide Generation Guide) — ฉบับ Canva

> ใช้คู่กับ [`template_slides.md`](../template/template_slides.md) (โครง 10 สไลด์ — หัวข้ออะไรอยู่หน้าไหน) และ [`slide_content_generator.md`](slide_content_generator.md) (วิธีเขียนเนื้อหา/กฎตัดหน้า) — ไฟล์นี้บอก **จัดวางอย่างไร** ให้อ่านง่ายและดูเป็นมืออาชีพ และ **สร้างเป็นสไลด์จริงด้วย [Canva](https://www.canva.com)** ตั้งแต่ตั้งโปรเจกต์ → กรอกเนื้อหา → ซ้อม/นำเสนอ → แชร์และ export สำรอง (§8–§10)
>
> ไฟล์ "รายละเอียดรายหน้า" ต่อ module ที่กรอกเนื้อหาครบแล้วสำหรับใช้กับคู่มือนี้ อยู่ในโฟลเดอร์ `slide/` ของ repo (เช่น `slide/m8.md`, `slide/m9.md`) — แต่ละไฟล์มีข้อความที่ต้องพิมพ์ + prompt สำหรับ Canva ครบทุกหน้า

## 1. หลักการออกแบบโดยรวม (5 ข้อที่ห้ามผิด)

| หลักการ | ทำไมสำคัญ |
|---------|-----------|
| **1 สไลด์ = 1 ความคิดหลัก** | สไลด์ที่มี 2 หัวข้อในหน้าเดียวทำให้ผู้ฟังไม่รู้จะโฟกัสจุดไหน |
| **ข้อความน้อยกว่าที่พูด** | สไลด์คือตัวช่วยความจำผู้ฟัง ไม่ใช่สคริปต์อ่าน — ถ้าอ่านสไลด์แล้วเข้าใจหมดโดยไม่ต้องฟัง แปลว่าเนื้อหาเยอะเกินไป |
| **ภาพแทนคำเมื่อทำได้** | สมองประมวลภาพเร็วกว่าตัวอักษร โดยเฉพาะ diagram/ตาราง/ไอคอน — Canva มี Elements ให้ค้นใช้ได้ทันที |
| **ทุกสไลด์ใช้ template เดียวกัน** | ตำแหน่งหัวข้อ โลโก้ เลขหน้า ต้องอยู่จุดเดิมทุกหน้า — ใน Canva ทำ "หน้าแม่แบบ" หน้าเดียวแล้ว Duplicate ทุกหน้าจากมัน (§8.2) อย่าเริ่มหน้าใหม่จาก template คนละอันปนกัน |
| **ผู้ฟังต้องรู้เสมอว่าอยู่ตรงไหนของเรื่อง** | หน้า Outline + เลขหัวข้อกำกับทุกสไลด์ + footer (หัวข้อปัจจุบัน/เลขหน้า) ทำให้กรรมการตามทัน จดอ้างอิง และย้อนถามได้แม่นยำ (วิธีทำทั้งหมดอยู่ §8.4) |

## 2. กฎความหนาแน่นของข้อความ (Text Density)

ใช้กฎ **5x5 แบบไทย** (ปรับจาก 6x6 สากล เพราะตัวอักษรไทยกว้างกว่าอังกฤษ) **บวกเกณฑ์จำนวนคำ** เพื่อกะปริมาณเนื้อหาให้พอดีกับพื้นที่สไลด์ตั้งแต่ตอนเขียนสคริปต์ ไม่ต้องรอไปเห็นตอนพิมพ์ลง Canva แล้วล้น:

- **ไม่เกิน 5 bullet ต่อสไลด์**
- **ไม่เกิน 1 บรรทัดต่อ bullet** เมื่อฉายจริง **และไม่เกิน 8–10 คำต่อ 1 bullet** (นับคำภาษาไทยแบบคำต่อคำ ไม่ใช่พยางค์) — ตัวเลขนี้คือความยาวที่ฟอนต์ Body 20–24 pt (เกณฑ์ §4) มักพอดี 1 บรรทัดบนสไลด์ 16:9 ถ้ายาวกว่านั้นตอนแก้ไข ให้ตัดคำ ไม่ใช่ลดขนาดฟอนต์
- **Title ไม่เกิน 6–8 คำ, Subtitle ไม่เกิน 12–15 คำ** (ทั้งคู่ต้องจบใน 1 บรรทัดเสมอ)
- **รวมทั้งสไลด์ (ไม่นับ Title/Subtitle) ไม่ควรเกิน ~40 คำ** — ถ้าเนื้อหาจำเป็นต้องเกินตัวเลขนี้ แปลว่าสไลด์นั้นมี 2 ความคิดซ้อนกันอยู่ ให้แตกหน้าตามข้อถัดไป ไม่ใช่ยัดเพิ่ม
- **ไม่มีย่อหน้ายาว** — ถ้าต้องอธิบายเยอะ ให้ย้ายไปช่อง **Notes** ของ Canva (ช่อง "รายละเอียดประกอบ" ใน template_slides.md)
- ถ้าเนื้อหาไม่พอในกฎนี้ → **แตกเป็น 2 สไลด์** (Duplicate page แล้วแบ่งเนื้อหา) อย่ายัดใส่หน้าเดียวโดยลดฟอนต์ให้เล็กลง
- ระวังนิสัยของ Canva: กล่องข้อความจะ**หดฟอนต์อัตโนมัติ**เมื่อพิมพ์ล้นกล่อง — ถ้าเห็นตัวหนังสือเล็กลงเอง แปลว่าเนื้อหาล้นแล้ว ให้ตัดคำหรือตัดหน้า **ห้ามปล่อยให้ฟอนต์หดต่ำกว่าเกณฑ์ §4**

## 3. หลักการจัดวาง Layout และตำแหน่งอย่างมืออาชีพ

### 3.1 หลักการทั่วไปที่ใช้กับทุกสไลด์ (ดูก่อนเปิดตาราง §3.2)

ไม่ว่าจะเป็น layout แบบไหนในตาราง §3.2 ให้ยึด 5 หลักการออกแบบนี้เสมอ — ขาดข้อไหนไปสไลด์จะดู "มือสมัครเล่น" ทันทีแม้เนื้อหาจะถูกต้องครบ:

| หลักการ | ทำอย่างไรใน Canva |
|---------|-------------------|
| **Alignment (จัดชิดแนวเดียวกัน)** | ทุกองค์ประกอบ (text box, การ์ด, ภาพ, ไอคอน) ต้องชิดแนวเดียวกันอย่างน้อย 1 แกน (ซ้าย/กลาง/บน) — เปิดใช้เส้น snap สีชมพูที่ Canva ขึ้นอัตโนมัติเมื่อลากใกล้แนวองค์ประกอบอื่น หรือใช้ `Position → Align` จัดหลายชิ้นให้ตรงแนวเดียวกันทีเดียว ห้ามมีองค์ประกอบ "เกือบตรง" |
| **Proximity (จัดกลุ่มสิ่งที่เกี่ยวข้อง)** | สิ่งที่สัมพันธ์กันต้องอยู่ใกล้กันกว่าสิ่งที่ไม่เกี่ยวข้อง — ระยะห่างระหว่าง bullet ในหัวข้อเดียวกันต้อง**น้อยกว่า**ระยะห่างระหว่างกลุ่ม/การ์ดคนละหัวข้อเสมอ ไม่งั้นตาจะแยกกลุ่มผิด |
| **Contrast / Hierarchy (ลำดับความสำคัญ)** | ให้ตาเห็นว่าอะไรสำคัญสุดก่อนโดยไม่ต้องอ่าน ผ่านขนาด/น้ำหนักฟอนต์/สี ตามลำดับ Title (ใหญ่+หนาสุด) → Subtitle → Body — ใช้ตัวหนา/สีเน้นไม่เกิน 1 จุดต่อสไลด์ ถ้าเน้นทุกอย่างเท่ากับไม่มีอะไรเด่นเลย |
| **White space (พื้นที่ว่าง)** | เว้นขอบสไลด์อย่างน้อย **5–8% ของความกว้าง** ทุกด้าน (สไลด์ 1920×1080 ≈ เว้นขอบ 100–150 px) และพื้นที่ที่มีองค์ประกอบจริงไม่ควรเกิน ~70% ของหน้า — พื้นที่ว่างคือสิ่งที่ทำให้สไลด์ดูเป็นมืออาชีพ ไม่ใช่ที่ว่างที่ต้อง "เติมให้เต็ม" |
| **Visual flow (ลำดับการมอง)** | คนอ่านตามแนว **Z-pattern**: ซ้ายบน → ขวาบน → ทแยงลงมาซ้ายล่าง → ขวาล่าง — วาง Title ไว้ซ้ายบนเสมอ, จุดเน้น/ภาพหลักอยู่กลาง-ขวา ถ้ามีภาพคู่กับข้อความ ให้ตัวแบบ/ลูกศรในภาพหันเข้าหาข้อความ ไม่ใช่หันออกนอกขอบสไลด์ |

### 3.2 Layout Pattern แนะนำต่อสไลด์ (อ้างอิงเลขสไลด์ตาม template)

> เลขสไลด์อ้างอิงตาม [`template_slides.md`](../template/template_slides.md) (10 สไลด์)
> คอลัมน์ "วิธีทำใน Canva" = องค์ประกอบที่ใช้ประกอบหน้านั้น (เมนู Elements / Text / Uploads)

| # | สไลด์ | Layout ที่แนะนำ | วิธีทำใน Canva | กฎเฉพาะ |
|---|-------|-----------------|----------------|---------|
| 1 | หน้าปก | **Title Slide** — ชื่อโครงงานกลางจอ, โลโก้มุมบน/ล่าง, ชื่อผู้จัดทำใต้ชื่อโครงงาน | Text box กึ่งกลาง + Uploads (โลโก้มหาวิทยาลัย) | ห้ามใส่ bullet ใดๆ ในหน้านี้ |
| 1.5 | Outline (หน้าแสดงหัวข้อ) | **Numbered Agenda** — รายการหัวข้อ 1–9 เรียงตามลำดับนำเสนอ | Text box + เลขในวงกลม (Elements → ค้น "circle number" หรือวาด Circle + ตัวเลข) | แทรกหลังหน้าปก 1 หน้า — เลขหัวข้อในหน้านี้คือเลขอ้างอิงที่ต้องปรากฏหน้าชื่อทุกสไลด์ถัดไป (§8.4) |
| 2 | ที่มาและเหตุผล | **Title + Bullet 3 ชั้น** (บริบท → ปัญหา → ผลกระทบ) มีไอคอนหน้า bullet | Bullet list + Elements (ค้นไอคอน "warning", "building", "impact") | ใช้ลำดับเดียวกับ 3 ย่อหน้าใน proposal_ch1.md ข้อ 1.1 |
| 3 | Software Survey | **ตารางเปรียบเทียบ** สูงสุด 4 คอลัมน์ (รวมคอลัมน์ "ระบบที่เสนอ") | Elements → **Table** แล้วปรับจำนวนแถว/คอลัมน์ | ไฮไลต์คอลัมน์ระบบที่เสนอด้วยสีพื้น cell ต่างจากคอลัมน์อื่น ให้เด่นสุด |
| 4 | วัตถุประสงค์ | **Numbered List** พร้อม number badge วงกลม (1,2,3...) | Text box + วงกลมเลขจาก Elements | ไม่เกิน 4 ข้อต่อสไลด์ ถ้าเกินแตกหน้า |
| 5 | ขอบเขตโครงงาน | **Multi-column by Role** สูงสุด 3-4 คอลัมน์เท่ากัน | การ์ด = Rectangle (มุมโค้ง) + Text box ซ้อน — ทำใบแรกให้เสร็จแล้ว **Group + Duplicate** ให้ทุกใบเท่ากันเป๊ะ | role เกิน 4 ให้แตกเป็น 2 สไลด์ ไม่บีบคอลัมน์แคบลง |
| 6 | เครื่องมือที่ใช้พัฒนา | **Icon/Logo Grid** จัดกลุ่มเป็นแถวตามหมวด (Frontend/Backend/DB/Infra) | Elements → ค้นชื่อเทคโนโลยี (react, python, postgresql, docker …) หรือ Uploads โลโก้ทางการ | ใช้โลโก้จริงของเทคโนโลยีแทนชื่อตัวหนังสือเมื่อทำได้ |
| 7 | System Overview | **Diagram-centric** — ภาพสถาปัตยกรรมกินพื้นที่ ≥ 70% ของสไลด์ | Uploads ภาพ diagram ที่ export ไว้ หรือวาดในหน้าเลย (Elements → Shapes + Lines/Arrows) | ข้อความอธิบายให้อยู่ขอบสไลด์เป็น label สั้นๆ ไม่ใช่ย่อหน้า — label บนภาพ ≥ 16 pt |
| 8 | Functional Requirements | **ตารางสรุปตาม Epic** ไม่เกิน 6-7 แถวต่อสไลด์ | Elements → Table | Epic เกิน 7 ให้แตกเป็น 2 สไลด์ อย่าลดฟอนต์จนอ่านไม่ออก |
| 9 | แผนการพัฒนา 3 เฟส | **3-Column Timeline/Kanban** (หลัก \| รอง \| ที่เหลือ) | การ์ด 3 ใบ (Group + Duplicate) หัวการ์ดสีเขียว/เหลือง/แดง | ใช้สีไล่ระดับความสำคัญให้สอดคล้องทั้ง 3 คอลัมน์ |
| 10 | Demo Mockup | **Image-heavy** — screenshot/mockup เต็มพื้นที่ พร้อม caption สั้นใต้ภาพ | Uploads screenshot จริง + วงกลมเลข ①②③ + Elements → Arrow เชื่อมลำดับ + ใส่ **Link** บนภาพชี้ URL mockup ที่ deploy แล้ว (คลิกได้ตอน Present) | ถ้าโชว์หลายหน้าจอต่อกัน ใช้ลูกศรเลข 1→2→3 กำกับลำดับการใช้งาน — 1 หน้า = 1 flow |

> **เครื่องมือ Canva ที่ควรรู้จัก:** `Position → Align` (จัดตำแหน่งให้ตรงกันทุกหน้า), `Group` (รวมการ์ด/ภาพ+caption ให้ย้ายพร้อมกัน), คลิกขวา → `Copy style` (ก๊อปฟอนต์+สีไปป้ายองค์ประกอบอื่น), `Lock` (ล็อกองค์ประกอบ footer กันเผลอลาก), Grid view (ดูทุกหน้าพร้อมกัน — ลากสลับลำดับหน้าได้)

## 4. กฎด้านตัวอักษร (Typography)

| ระดับ | ขนาดฟอนต์ขั้นต่ำ (ฉายจอโปรเจกเตอร์) | ตัวอย่างการใช้ |
|-------|----------------------------------|----------------|
| หัวข้อสไลด์ (Title) | 32–40 pt | ชื่อสไลด์บนสุด |
| เนื้อหาหลัก (Body) | 20–24 pt | bullet, ตาราง |
| หมายเหตุ/แหล่งอ้างอิง | ≥ 14 pt | ระบุแหล่งข้อมูลมุมล่างสไลด์ (ถ้าใส่) |

**ฟอนต์ที่แนะนำ (รองรับภาษาไทยครบ ไม่มีปัญหาสระ/วรรณยุกต์หาย):**
- อันดับแรก: **`Sarabun`** — มีให้เลือกใน Canva โดยตรง โครงสร้างเดียวกับ TH Sarabun New ที่ใช้ในเอกสาร proposal/fn req (ตัว TH Sarabun New เองไม่มีใน Canva Free — ถ้ามี Canva Pro อัปโหลดไฟล์ฟอนต์เองได้ผ่าน Brand Kit)
- ทางเลือก: `Kanit`, `Noto Sans Thai`, `Prompt` (มีใน Canva ทั้งหมด)
- **ห้ามใช้ฟอนต์ที่ไม่รองรับภาษาไทย** (ฟอนต์ละตินล้วน) — Canva จะ fallback ไปฟอนต์อื่นเงียบๆ ทำให้หน้าตาไม่ตรงกันระหว่างหน้า ตรวจด้วยตาทุกหน้าว่าสระบน/ล่างไม่เยื้อง
- **ตั้งครั้งเดียวใช้ทั้ง deck:** กำหนดฟอนต์บน "หน้าแม่แบบ" (§8.2) แล้ว Duplicate — อย่าเลือกฟอนต์ใหม่รายหน้า และอย่าใช้ฟอนต์เกิน 1 ตระกูลทั้ง deck

**กฎการตัดบรรทัดภาษาไทย:** Canva ตัดบรรทัดที่ช่องว่างเท่านั้น — ประโยคไทยยาวๆ ต้อง**เคาะวรรคเองตรงจุดที่ควรตัดคำ** แล้วลากขอบกล่องข้อความให้ตัดตรงตำแหน่งที่ต้องการ ห้ามปล่อยให้คำขาดกลาง

## 5. กฎด้านสี (Color System)

- **สีหลักไม่เกิน 2 สี + สีกลาง** (ขาว/เทา/ดำ) — ยึดสีของมหาวิทยาลัย/คณะเป็นสีหลักถ้ามี
- **สัดส่วนการใช้สีแบบ 60-30-10** (กฎมาตรฐานงานออกแบบ กันสีแย่งความสนใจกันเอง): พื้นหลัง/พื้นที่ว่างโทนกลาง (ขาว/เทาอ่อน) ~60% + สีรองสำหรับหัวข้อ/กล่อง/กราฟ ~30% + สีเน้น (accent) เฉพาะจุดสำคัญที่สุดของหน้านั้น ~10%
- **ความคมชัดของข้อความต้องอ่านออกจากที่นั่งหลังห้อง:** ใช้คู่สีตัวอักษร/พื้นหลังที่ contrast สูง (ตัวเข้มบนพื้นอ่อน หรือกลับกัน) หลีกเลี่ยงสีอ่อนบนสีอ่อนใกล้เคียงกัน (เช่น เทาอ่อนบนขาว) — ถ้าไม่แน่ใจ เช็คด้วย contrast checker ตามเกณฑ์ WCAG ให้ได้อย่างน้อยระดับ AA
- **ใช้สีสื่อความหมายให้สม่ำเสมอทุกสไลด์** สอดคล้องกับที่ใช้ในเอกสาร fn req/usecase:

| ความหมาย | สีแนะนำ | ใช้ที่ไหน |
|----------|---------|-----------|
| ทำงานเสร็จ/ใช้งานจริง | เขียว | Slide 8-9 (ฟังก์ชันที่ active) |
| ยังไม่พัฒนา (dev in next version) | เทา | Slide 5, 8-9 (ฟังก์ชัน 🔜) |
| จุดเน้น/ระบบที่เสนอ | สีหลักของธีม (เข้ม) | Slide 3 (คอลัมน์ระบบที่เสนอ), Slide 9 (ฟังก์ชันหลัก) |
| คำเตือน/ข้อควรระวัง | ส้ม/เหลือง | จุดที่ซับซ้อน (⚠ ตาม fn req) |

- **วิธีคุมสีใน Canva:** พิมพ์รหัส hex ตรงๆ ในตัวเลือกสี (เช่น `#1A56DB`) แล้ว Canva จะจำไว้ใน "Recently used" / Document colors ให้เลือกซ้ำ — ถ้ามี Canva Pro ตั้งเป็น **Brand Kit** ครั้งเดียวใช้ได้ทุกไฟล์
- หลีกเลี่ยงพื้นหลังไล่สี (gradient) เข้มเกินไปที่ทำให้ข้อความอ่านยาก โดยเฉพาะเมื่อฉายในห้องที่มีแสงสว่าง — template สวยๆ ของ Canva จำนวนมากพื้นเข้ม/gradient จัด **เลือก template โทนขาวสะอาดเป็นฐานแล้วค่อยแต่ง** ง่ายกว่าแก้ template เข้มให้อ่านออก

## 8. การสร้างสไลด์ด้วย Canva (Content → Presentation)

### 8.1 เริ่มโปรเจกต์ — เลือก 1 จาก 3 ทาง

สร้างงานใหม่ประเภท **Presentation (16:9)** ตั้งชื่อไฟล์ `slides-<module>` (เช่น `slides-m9`) แล้วเลือกวิธีขึ้นโครง:

| วิธี | เหมาะกับ | ขั้นตอน |
|------|----------|---------|
| **(ก) Magic Design™** (แนะนำ — เร็วสุด) | มีไฟล์รายละเอียดรายหน้าใน `slide/<module>.md` แล้ว | หน้า Home → ช่องสร้างงาน → วาง **"Prompt หลัก"** จากท้ายไฟล์ `slide/<module>.md` → เลือกแบบที่โทนขาวสะอาด → ได้โครงทุกหน้า แล้วแทนที่ข้อความด้วยส่วน "ข้อความที่ต้องพิมพ์" ของแต่ละหน้า |
| **(ข) Canva Docs → Convert** | ถนัดเขียนเนื้อหาเป็นเอกสารก่อน | สร้าง Canva Doc → วางเนื้อหาทีละหัวข้อ (ใช้ Heading คั่นหน้า) → ปุ่ม Convert to Presentation → Canva แตกหน้าให้ตามหัวข้อ |
| **(ค) เริ่มจาก template เปล่า** | อยากคุมทุกอย่างเอง | เลือก template presentation โทนขาว minimal → ลบหน้าที่ไม่ใช้ → ทำหน้าแม่แบบตาม §8.2 แล้วสร้างทีละหน้า |

> ไม่ว่าทางไหน ผลลัพธ์ที่ต้องได้เท่ากัน: จำนวนหน้า + หัวข้อ + เนื้อหา ตรงตามไฟล์ `slide/<module>.md` (หรือ `slides_outline.md`) ทุกหน้า — Magic Design เป็นแค่ตัวขึ้นโครง อย่าปล่อยให้มันเปลี่ยนโครงเนื้อหา

### 8.2 ตั้งค่าทั้ง deck ครั้งเดียว — "หน้าแม่แบบ"

Canva ไม่มี master slide แบบ PowerPoint — ใช้วิธีนี้แทน:

1. จัดหน้าเนื้อหาหน้าแรกให้สมบูรณ์ที่สุด 1 หน้า: **Title** (ตำแหน่ง+ขนาดตายตัว ซ้ายบน 36–40 pt หนา) → **Subtitle** ใต้ Title (20–22 pt สีเทา) → **footer** (ซ้ายล่าง = ที่ว่างสำหรับชื่อหัวข้อ, ขวาล่าง = ที่ว่างสำหรับเลขหน้า, 14 pt)
2. **Lock** องค์ประกอบ footer (คลิกขวา → Lock) กันเผลอลากตอนแก้เนื้อหา
3. ทุกหน้าใหม่ให้ **Duplicate จากหน้านี้** แล้วแก้เฉพาะเนื้อหากลางหน้า — ห้ามสร้างหน้าเปล่าแล้วจัดใหม่ (ตำแหน่งจะเพี้ยนทีละนิดจนดูไม่เป็นมืออาชีพ)
4. ฟอนต์ (Sarabun) และชุดสี (พิมพ์ hex ตาม design system ของ `slide/<module>.md`) กำหนดที่หน้านี้ให้ครบก่อน Duplicate

### 8.3 การ map เนื้อหาจากไฟล์รายละเอียดรายหน้า → Canva

แต่ละช่องในไฟล์ `slide/<module>.md` (หรือช่องใน `slides_outline.md` ตาม [`template_slides.md`](../template/template_slides.md)) มีที่ลงตายตัวใน Canva — ไม่ต้องตีความใหม่:

| ช่องในไฟล์รายละเอียด | ไปอยู่ที่ไหนใน Canva |
|----------------------|----------------------|
| ชื่อหน้า เช่น "1 · ที่มาและเหตุผล" | Text box **Title** ของหน้า (จากหน้าแม่แบบ) |
| Subtitle (ประโยคใจความหลัก) | Text box ใต้ Title |
| ข้อความที่ต้องพิมพ์ (bullet/ตาราง) | Bullet = Text box แบบ list · ตาราง = Elements → **Table** |
| Script พูด / Speaker Notes | ช่อง **Notes** ใต้หน้านั้น (ปุ่ม Notes มุมล่างซ้ายของ editor) — โชว์เฉพาะใน Presenter view ผู้ชมไม่เห็น |
| Layout Canva ที่ระบุต่อหน้า | จัดองค์ประกอบตามนั้น (การ์ด/grid/ภาพเต็มหน้า — เทคนิคดูตาราง §3) |
| หน้าที่ตัดเพิ่ม เช่น "(1/2)" (กฎใน [`slide_content_generator.md`](slide_content_generator.md) §4) | **Duplicate page** เดิม → เปลี่ยนเลขใน Title เป็น (2/2) → เปลี่ยนเฉพาะเนื้อหา ส่วน layout/ฟอนต์คงเดิมทั้งหมด |
| รูปภาพ (diagram/screenshot mockup/โลโก้) | **Uploads** → ลากลงหน้า — screenshot ควร capture ที่ความละเอียดเต็มจอ ไม่ย่อก่อนอัป |
| Prompt หน้านี้ (ในไฟล์ `slide/<module>.md`) | ใช้กับ Magic Design ตอนขึ้นโครง หรือใช้ปรับหน้าเดี่ยวภายหลัง |

### 8.4 องค์ประกอบนำทาง — Outline, เลขหัวข้อ, Title/Subtitle, footer เลขหน้า

เป้าหมาย: ผู้ฟังตอบได้ตลอดเวลาว่า "ตอนนี้อยู่หัวข้อไหน และเหลืออีกแค่ไหน" — ใช้ 4 กลไกนี้ร่วมกัน

**(1) เลขหัวข้อ 1–9 ผูกกับหน้า Outline** — เลขชุดเดียวใช้ทั้ง deck:

| หัวข้อ # | ชื่อหัวข้อ | คือ Slide # ใน template_slides.md |
|:---:|-----------|:---:|
| 1 | ที่มาและเหตุผล | 2 |
| 2 | Software Survey | 3 |
| 3 | วัตถุประสงค์ | 4 |
| 4 | ขอบเขตโครงงาน | 5 |
| 5 | เครื่องมือที่ใช้พัฒนา | 6 |
| 6 | System Overview | 7 |
| 7 | Functional Requirements | 8 |
| 8 | แผนการพัฒนา 3 เฟส | 9 |
| 9 | Demo Mockup | 10 |

- ชื่อทุกสไลด์เนื้อหาขึ้นต้นด้วยเลขหัวข้อ: `4 · ขอบเขตโครงงาน` — สะกดตรงกับหน้า Outline คำต่อคำ
- หน้าที่ตัดเพิ่ม: เลขเดิม + เลขหน้าย่อย `4 · ขอบเขตโครงงาน (2/2)`

**(2) Title + Subtitle ทุกหน้าเนื้อหา** — 2 ชั้นเสมอ ห้ามมีแค่ชั้นเดียว:

- **Title** = เลขหัวข้อ + ชื่อหัวข้อ (ตรงกับ Outline)
- **Subtitle** = ประโยคใจความหลักของหน้านั้น 1 บรรทัด (= "ประโยคเดียวที่ผู้ฟังต้องจำ" ตาม [`slide_content_generator.md`](slide_content_generator.md) §1) — หน้าแตกเพิ่มใช้ Title เดิมแต่ Subtitle เปลี่ยนตามใจความของหน้านั้น

**(3) หน้า Outline** — แทรก 1 หน้าหลังหน้าปก เป็น numbered list หัวข้อ 1–9 ตอนพูดใช้เวลาแค่ ~15 วินาที — เลขชุดนี้คือ "สารบัญ" ที่กรรมการใช้จดอ้างอิงตอนถาม

**(4) footer: หัวข้อปัจจุบัน + เลขหน้า (ทำบนหน้าแม่แบบครั้งเดียว)**

Canva ไม่มีเลขหน้า/แถบ progress อัตโนมัติแบบโปรแกรมสไลด์ทั่วไป — ใช้วิธี manual ที่คุมง่าย:

- **ซ้ายล่าง** = ชื่อหัวข้อปัจจุบัน เช่น `หัวข้อ 4/9 · ขอบเขตโครงงาน` — Duplicate หน้าแล้วแก้ข้อความนี้ให้ตรงหัวข้อ (หน้าแตกเพิ่มใช้ข้อความเดิมซ้ำ)
- **ขวาล่าง** = เลขหน้า `ปัจจุบัน / ทั้งหมด` เช่น `7 / 13` — **กรอกเป็นขั้นตอนสุดท้าย**หลังจำนวนหน้านิ่งแล้ว (ตัดหน้าเพิ่ม/ลบหน้าจบก่อน ค่อยไล่กรอกทีเดียวจาก Grid view จะไม่พลาด)
- **progress bar (ทางเลือก)** = Rectangle บางๆ ชิดขอบล่าง ความกว้าง = (เลขหน้า ÷ จำนวนหน้าทั้งหมด) × ความกว้างสไลด์ — สวยแต่ต้องปรับมือทุกหน้า ถ้าเวลาน้อยตัดทิ้งได้ เลขหน้า `7 / 13` สื่อความคืบหน้าได้เพียงพอแล้ว

### 10.1 ลิงก์สำหรับกรรมการ/อาจารย์ (แบบดูอย่างเดียว)

1. ปุ่ม **Share** → Anyone with the link → **Can view** → Copy link
2. ต้องการให้เปิดมาเป็นโหมดนำเสนอเลย: Share → **More → Present** (ได้ลิงก์ที่เปิดแบบเต็มจอ ไม่เห็นหน้าตา editor)
3. **กรอก URL นี้กลับเข้าไปในช่อง "URL Slides"** ของ `slides_outline.md` / ตาราง Reference ในไฟล์ `slide/<module>.md` (convention เดียวกับที่ template กำหนด)

### 10.2 ข้อควรระวังเรื่องลิงก์ Canva

- ลิงก์ Canva **ต้องมีอินเทอร์เน็ต**เสมอ — วันสอบให้พึ่ง PDF สำรอง (§9) เป็นแผนสอง ห้ามพึ่งลิงก์อย่างเดียว
- ลิงก์ผูกกับบัญชี Canva ของเจ้าของไฟล์ — อย่าลบไฟล์/ย้ายทีมหลังส่งลิงก์แล้ว
- ภาพ mockup บนสไลด์เป็น screenshot นิ่ง ส่วน **ลิงก์ไป mockup จริง** (GitHub Pages ของ repo) ใส่เป็น Link บนภาพ/ปุ่มในสไลด์ — เช็คว่า URL mockup ยังเปิดได้ก่อนวันสอบ

### 10.3 ส่งไฟล์เข้า repo (convention ของ repo นี้)

สไลด์ Canva ไม่มี source ในรูปไฟล์ข้อความ — สิ่งที่เก็บเข้า repo คือ:

- ไฟล์รายละเอียดรายหน้า `slide/<module>.md` (เป็น source of truth ของเนื้อหา — แก้เนื้อหาให้แก้ที่นี่ก่อนแล้วค่อยไปแก้ใน Canva)
- PDF ที่ export แล้ว วางใน `slide/` เช่น `slide/slides-m9.pdf` (ไว้เปิดดูได้โดยไม่ต้องเข้า Canva)
- URL ลิงก์ Canva กรอกในตาราง Reference ของไฟล์รายละเอียดรายหน้า
